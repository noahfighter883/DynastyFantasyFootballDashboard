# DynastyEvaluator

A dynasty fantasy football analytics dashboard that answers the question every dynasty manager eventually asks: *if my league re-drafted from scratch today, how would every team actually stack up?*

**[Live app →](https://dynasty-fantasy-football-dashboard.vercel.app/)**

Built for a real 12-team dynasty league, pulling live roster, ranking, and transaction data to compare all 12 teams across three lenses: long-term dynasty value, this-year redraft value, and projected points.

---

## Features

- **League Overview** — all 12 teams ranked against each other, toggleable by dynasty value, redraft value, or projected points, and by starters, "starters +1," or full roster
- **Position Comparison** — isolate a single position (QB/RB/WR/TE) and rank every team's strength at just that spot
- **Team Detail** — full roster breakdown, position-by-position cards, and a custom **Draft Capital Curve** chart plotting every player's dynasty and redraft rank, sorted best to worst
- **Acquired tracking** — every player is tagged as Drafted, Traded for, or picked up on Waivers, sourced from the league's full multi-season transaction history
- **"Starters +1"** — a dynamic view showing each team's real starters plus the single best bench player per position, re-selected depending on which value lens you're viewing

## How it works

A Python pipeline pulls and joins data from several sources:

- **[Sleeper API](https://docs.sleeper.com/)** — live rosters, starters, league scoring settings, and full multi-season draft/transaction history
- **[DynastyProcess](https://github.com/dynastyprocess/data)** — open dynasty and redraft player rankings, including positional ranks
- Season-long **projected points**, calculated directly from the league's actual scoring settings (including a TE premium)

Team-level "value" is an **average-rank system** (lower average rank = better team — the same logic as golf scoring) rather than a raw sum, so a team's score reflects player quality regardless of roster size.

The frontend is a React app that consumes the joined dataset and renders the three main views described above, including a hand-built SVG chart (no charting library) for the Draft Capital Curve.

## Tech stack

- **Data pipeline:** Python
- **Frontend:** React, TypeScript
- **Hosting:** Vercel
- Built with [Figma Make](https://www.figma.com/make/), with development assistance from [Claude](https://claude.ai) (Anthropic)

## Data freshness

The dataset is a snapshot — regenerated by re-running the Python pipeline and updating the frontend's data file, not a live database connection.
